# Layout Engine

PP-DOS organizes products around workspaces.

## Shared Layout Rules

- Use constrained content widths for reading and wide grids for dashboards.
- Keep cards at 8px radius unless a component has a specific reason.
- Avoid cards inside cards except repeated records or state panels.
- Put priority intelligence above lower-level tables.
- Keep trust indicators visible near claims, scores, predictions, and simulations.

## Workspace Layouts

### Fan Workspace

Fast, scannable, personalized. Optimized for Command Center, watchlists, rankings, player/team intelligence, TMX, and maps.

### Coach Workspace

Dense, operational, private-ready. Optimized for Atlas, roster simulation, Transfer Fit, Scholarship IQ, staff notes, and recruiting boards.

### GM Workspace

Strategic planning layout. Optimized for multi-year forecasts, Digital Twin, roster construction, position risk, and conference comparison.

### Media Workspace

Research-forward layout. Optimized for source timelines, exportable summaries, historical replay, player/team context, and relationship graphs.

### Admin Workspace

Queue-first layout. Optimized for claim review, conflicts, source checks, duplicate resolution, NIL review, and audit trails.

### Developer Workspace

Documentation and testing layout. Optimized for endpoints, schemas, examples, webhooks, usage, and API trust rules.
