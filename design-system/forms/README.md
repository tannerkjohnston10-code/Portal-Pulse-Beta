# Form Standards

Forms in Portal Pulse should be clear, accessible, validated, and respectful of user attention.

## Principles

- Use labels for every input.
- Show validation near the affected field.
- Preserve user input when errors occur.
- Avoid unnecessary required fields.
- Use placeholders as hints, not labels.
- Group related fields into panels or fieldsets.

## Controls

- Search inputs for discovery.
- Select menus for known option sets.
- Segmented controls for small mutually exclusive choices.
- Checkboxes and toggles for binary settings.
- Sliders or steppers for numeric values when precision is not typed.
- Textareas for notes and long input.

## Trust And Safety

Forms that create claims, reviews, simulations, API keys, or admin changes must show impact, confidence, permissions, and confirmation where appropriate.

## Accessibility

Inputs must support keyboard use, visible focus, error announcements, and touch targets of at least 44px for primary controls.
