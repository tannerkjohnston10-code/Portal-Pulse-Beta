# PP-DOS Documentation Guide

PP-DOS documentation should explain visual intent, interaction behavior, accessibility, responsiveness, component rules, and product trust requirements.

Every new design system addition should document:

- purpose
- user problem
- component anatomy
- states
- accessibility
- motion
- theme behavior
- trust behavior
- examples
- anti-patterns

Documentation should be product-readable and engineering-actionable.

## Canonical Files

- Master design reference: `design-system/PORTAL_PULSE_DESIGN_OS.md`
- Token contract: `design-system/tokens/pp-dos-tokens.ts`
- Component catalog: `design-system/components/COMPONENT_CATALOG.md`
- Accessibility: `design-system/accessibility/ACCESSIBILITY_STANDARDS.md`
- Motion: `design-system/motion/MOTION_SYSTEM.md`
