# Motion System

Motion in PP-DOS must communicate meaning.

## Motion Principles

- Motion explains change.
- Motion should be fast and calm.
- Motion should never hide information.
- Motion should respect reduced-motion preferences.

## Patterns

### Rankings

Rank movement should animate position changes and delta color briefly. Avoid looping animation.

### Confidence Changes

Confidence changes should transition border/surface color and display a textual state change.

### Player Movement

Player movement should animate from origin to destination on maps or timelines only when it clarifies transfer flow.

### Timeline Progression

Timeline scrubbers should move deliberately and preserve readable event labels.

### Map Transitions

Map layer toggles should fade or slide with clear legends.

### Graph Expansion

Graph nodes should expand from the selected entity and maintain relationship labels.

### Loading

Skeletons should pulse subtly. Long-running AI placeholders should show what is being analyzed.

### Hover States

Hover should reveal affordance, not distract.

### Panel Transitions

Drawers and popovers should move from the edge or anchor that triggered them.

## Reduced Motion

When reduced motion is enabled, use instant state changes, static pagination, and non-animated progress indicators.
