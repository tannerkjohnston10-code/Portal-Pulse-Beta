# Technical Bible Index

The Technical Bible is the engineering architecture reference for Portal Pulse. Start with `PORTAL_PULSE_TECHNICAL_ARCHITECTURE.md`, then use the folders below for future deep dives.

- `architecture/` — system architecture, boundaries, diagrams, and decision records.
- `data/` — data model, claims, evidence, history, and trust architecture.
- `platform/` — Platform SDK, Intelligence Core, event bus, cache, and shared services.
- `ai/` — Athena Intelligence Network, agent lifecycle, safety, and evaluation.
- `apis/` — API contracts, serializers, versioning, rate limits, and developer platform.
- `infrastructure/` — deployment topology, queues, cache, storage, observability, and CI/CD.
- `security/` — auth, authorization, secrets, privacy, audit logs, and compliance.
- `testing/` — unit, integration, contract, accessibility, performance, and regression testing.
- `operations/` — release, incident, review, admin, and reliability operations.
- `future/` — enterprise, mobile, Digital Twin, Labs, and international expansion architecture.
