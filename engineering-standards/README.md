# Engineering Standards Index

Start with `ENGINEERING_STANDARDS.md`. This folder is the mandatory engineering rulebook for Portal Pulse contributors and AI coding assistants.

- `architecture/` — architecture rules, dependency boundaries, and modular ownership.
- `coding/` — TypeScript, React, Next.js, services, errors, and async patterns.
- `testing/` — unit, integration, contract, accessibility, performance, visual, and regression testing.
- `documentation/` — feature specs, architecture docs, rollout notes, and cross-documentation rules.
- `ui/` — PP-DOS integration, accessibility, responsive layouts, states, and motion.
- `apis/` — REST, versioning, envelopes, errors, pagination, auth, idempotency, and rate limits.
- `security/` — auth, authorization, secrets, privacy, audit logging, compliance, and dependency security.
- `performance/` — load, bundle size, lazy loading, caching, jobs, database efficiency, and observability.
- `reviews/` — pull request expectations, review culture, risk assessment, and approval standards.
- `operations/` — release, monitoring, incidents, admin workflows, and reliability operations.
- `checklists/` — PR checklist, Definition of Done, security review, accessibility review, and launch readiness.
